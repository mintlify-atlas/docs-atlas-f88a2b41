---
title: "Message schema: roles, content, and tool calls"
description: "Reference for schema.Message — roles, plain-text and multimodal content, tool calls, response metadata, and stream concatenation helpers."
---

`schema.Message` is the primary data type exchanged between chat models, prompt templates, tools, and your application code. Every message has a `Role` that identifies its author and a combination of text and structured fields that carry the actual content. Models that support multimodal input or output use `UserInputMultiContent` and `AssistantGenMultiContent` respectively instead of the plain `Content` string.

---

## Message struct

```go
type Message struct {
    Role                     RoleType           `json:"role"`
    Content                  string             `json:"content"`
    UserInputMultiContent     []MessageInputPart `json:"user_input_multi_content,omitempty"`
    AssistantGenMultiContent  []MessageOutputPart`json:"assistant_output_multi_content,omitempty"`
    Name                     string             `json:"name,omitempty"`
    ToolCalls                 []ToolCall         `json:"tool_calls,omitempty"`
    ToolCallID               string             `json:"tool_call_id,omitempty"`
    ToolName                 string             `json:"tool_name,omitempty"`
    ResponseMeta             *ResponseMeta      `json:"response_meta,omitempty"`
    ReasoningContent         string             `json:"reasoning_content,omitempty"`
    Extra                    map[string]any     `json:"extra,omitempty"`
}
```

<ResponseField name="Role" type="RoleType" required>
  Identifies who authored the message. One of `System`, `User`, `Assistant`, or `Tool`. See [RoleType](#roletype).
</ResponseField>

<ResponseField name="Content" type="string">
  Plain text content. Use this field for text-only messages. For multimodal user input, use `UserInputMultiContent` instead.
</ResponseField>

<ResponseField name="UserInputMultiContent" type="[]MessageInputPart">
  Multimodal content provided by the user. Each `MessageInputPart` can hold text, an image, audio, video, or a file. Use this field instead of `Content` when sending non-text media to the model.
</ResponseField>

<ResponseField name="AssistantGenMultiContent" type="[]MessageOutputPart">
  Multimodal content returned by the model. Each `MessageOutputPart` can hold text, an image, audio, video, or a reasoning block. Populated by the model; do not set manually.
</ResponseField>

<ResponseField name="Name" type="string">
  Optional display name for the author, used by some model providers to distinguish multiple participants with the same role.
</ResponseField>

<ResponseField name="ToolCalls" type="[]ToolCall">
  Present on `Assistant` messages when the model wants to invoke one or more tools. See [ToolCall](#toolcall).
</ResponseField>

<ResponseField name="ToolCallID" type="string">
  Present on `Tool` messages. Matches the `ID` from the corresponding `ToolCall` so the model can correlate results.
</ResponseField>

<ResponseField name="ToolName" type="string">
  Present on `Tool` messages. The name of the tool that produced this result.
</ResponseField>

<ResponseField name="ResponseMeta" type="*ResponseMeta">
  Model-level metadata attached to `Assistant` messages, including finish reason and token usage. See [ResponseMeta](#responsemeta).
</ResponseField>

<ResponseField name="ReasoningContent" type="string">
  The chain-of-thought or reasoning text produced by reasoning models (e.g. Claude or o1) before the final answer.
</ResponseField>

<ResponseField name="Extra" type="map[string]any">
  Arbitrary key-value pairs for model-specific extensions that do not fit the standard fields.
</ResponseField>

## RoleType

```go
type RoleType string

const (
    System    RoleType = "system"
    User      RoleType = "user"
    Assistant RoleType = "assistant"
    Tool      RoleType = "tool"
)
```

<ResponseField name="System" type="RoleType">
  A system prompt that sets the model's persona or instructions. Typically the first message in a conversation.
</ResponseField>

<ResponseField name="User" type="RoleType">
  A message from the human (or application acting on behalf of the human).
</ResponseField>

<ResponseField name="Assistant" type="RoleType">
  A message generated by the model. May include `ToolCalls` when the model requests tool execution.
</ResponseField>

<ResponseField name="Tool" type="RoleType">
  The result of a tool execution, sent back to the model so it can incorporate the output into its next response.
</ResponseField>

## ToolCall

```go
type ToolCall struct {
    Index    *int         `json:"index,omitempty"`
    ID       string       `json:"id"`
    Type     string       `json:"type"`
    Function FunctionCall `json:"function"`
    Extra    map[string]any `json:"extra,omitempty"`
}

type FunctionCall struct {
    Name      string `json:"name,omitempty"`
    Arguments string `json:"arguments,omitempty"`
}
```

<ResponseField name="Index" type="*int">
  Position of this tool call within the `ToolCalls` slice. Used when merging streaming chunks to identify which call a chunk belongs to.
</ResponseField>

<ResponseField name="ID" type="string">
  Unique identifier assigned by the model. Echoed back as `ToolCallID` on the corresponding `Tool` message.
</ResponseField>

<ResponseField name="Type" type="string">
  Always `"function"` for standard tool calls.
</ResponseField>

<ResponseField name="Function.Name" type="string">
  The name of the function to invoke.
</ResponseField>

<ResponseField name="Function.Arguments" type="string">
  JSON-encoded arguments for the function call.
</ResponseField>

<ResponseField name="Extra" type="map[string]any">
  Model-specific extra information.
</ResponseField>

## ResponseMeta

```go
type ResponseMeta struct {
    FinishReason string      `json:"finish_reason,omitempty"`
    Usage        *TokenUsage `json:"usage,omitempty"`
    LogProbs     *LogProbs   `json:"logprobs,omitempty"`
}
```

<ResponseField name="FinishReason" type="string">
  Why the model stopped generating. Common values: `"stop"`, `"length"`, `"tool_calls"`, `"content_filter"`. The exact set is model-specific.
</ResponseField>

<ResponseField name="Usage" type="*TokenUsage">
  Token counts for the request. Availability depends on the model provider.

  <Expandable title="TokenUsage fields">
    <ResponseField name="PromptTokens" type="int">
      Total input tokens consumed.
    </ResponseField>
    <ResponseField name="CompletionTokens" type="int">
      Total output tokens generated.
    </ResponseField>
    <ResponseField name="TotalTokens" type="int">
      Sum of prompt and completion tokens.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="LogProbs" type="*LogProbs">
  Token-level log-probability information, when requested and supported by the model.
</ResponseField>

## MessageInputPart

`MessageInputPart` represents one element in a multimodal user message.

```go
type MessageInputPart struct {
    Type  ChatMessagePartType `json:"type"`
    Text  string              `json:"text,omitempty"`
    Image *MessageInputImage  `json:"image,omitempty"`
    Audio *MessageInputAudio  `json:"audio,omitempty"`
    Video *MessageInputVideo  `json:"video,omitempty"`
    File  *MessageInputFile   `json:"file,omitempty"`
    Extra map[string]any      `json:"extra,omitempty"`
}
```

<ResponseField name="Type" type="ChatMessagePartType" required>
  Discriminator for the part. One of `"text"`, `"image_url"`, `"audio_url"`, `"video_url"`, `"file_url"`.
</ResponseField>

<ResponseField name="Text" type="string">
  Plain text. Set when `Type` is `"text"`.
</ResponseField>

<ResponseField name="Image" type="*MessageInputImage">
  Image data. Set when `Type` is `"image_url"`. Provide either `URL` or `Base64Data` (plus `MIMEType`).
</ResponseField>

<ResponseField name="Audio" type="*MessageInputAudio">
  Audio data. Set when `Type` is `"audio_url"`. Provide either `URL` or `Base64Data` (plus `MIMEType`).
</ResponseField>

<ResponseField name="Video" type="*MessageInputVideo">
  Video data. Set when `Type` is `"video_url"`. Provide either `URL` or `Base64Data` (plus `MIMEType`).
</ResponseField>

<ResponseField name="File" type="*MessageInputFile">
  File data. Set when `Type` is `"file_url"`. Supports an optional `Name` field in addition to `URL` / `Base64Data`.
</ResponseField>

```go
msg := &schema.Message{
    Role: schema.User,
    UserInputMultiContent: []schema.MessageInputPart{
        {Type: schema.ChatMessagePartTypeText, Text: "What is in this image?"},
        {
            Type: schema.ChatMessagePartTypeImageURL,
            Image: &schema.MessageInputImage{
                MessagePartCommon: schema.MessagePartCommon{
                    URL: ptr("https://example.com/cat.jpg"),
                },
                Detail: schema.ImageURLDetailHigh,
            },
        },
    },
}
```

## MessageOutputPart

`MessageOutputPart` represents one element in a multimodal model response.

```go
type MessageOutputPart struct {
    Type      ChatMessagePartType     `json:"type"`
    Text      string                  `json:"text,omitempty"`
    Image     *MessageOutputImage     `json:"image,omitempty"`
    Audio     *MessageOutputAudio     `json:"audio,omitempty"`
    Video     *MessageOutputVideo     `json:"video,omitempty"`
    Reasoning *MessageOutputReasoning `json:"reasoning,omitempty"`
    Extra     map[string]any          `json:"extra,omitempty"`
}
```

<ResponseField name="Type" type="ChatMessagePartType" required>
  One of `"text"`, `"image_url"`, `"audio_url"`, `"video_url"`, `"reasoning"`.
</ResponseField>

<ResponseField name="Text" type="string">
  Generated text. Set when `Type` is `"text"`.
</ResponseField>

<ResponseField name="Reasoning" type="*MessageOutputReasoning">
  Reasoning content from a reasoning model. Contains `Text` (the reasoning steps) and an optional `Signature` (encrypted reasoning token required by some APIs).
</ResponseField>

<ParamField path="content" type="string" required>

```go
)
```

## ConcatMessages

```go
func ConcatMessages(msgs []*Message) (*Message, error)
```

Merges a slice of messages that all share the same role into a single message. Useful for reassembling a streaming response after draining a `StreamReader[*Message]`.

- `Content` fields are concatenated.
- `ToolCalls` are merged by index, concatenating their `Arguments`.
- `ReasoningContent` fields are concatenated.
- `ResponseMeta.Usage` is taken as the maximum value seen across all chunks.
- Returns an error if any two messages have different `Role` or `Name` values.

<ParamField path="msgs" type="[]*Message" required>
  Ordered slice of message chunks from a stream. None may be `nil`.
</ParamField>

```go
var chunks []*schema.Message
for {
    chunk, err := sr.Recv()
    if errors.Is(err, io.EOF) { break }
    if err != nil { return err }
    chunks = append(chunks, chunk)
}

full, err := schema.ConcatMessages(chunks)
```

## ConcatMessageStream

```go
func ConcatMessageStream(s *StreamReader[*Message]) (*Message, error)
```

Convenience wrapper around `ConcatMessages` that drains an entire `StreamReader[*Message]` and closes it automatically.

<ParamField path="s" type="*StreamReader[*Message]" required>
  The stream to drain. The function calls `Close` before returning.
</ParamField>

```go
// sr is a *schema.StreamReader[*schema.Message] from runnable.Stream.
full, err := schema.ConcatMessageStream(sr)
if err != nil {
    return err
}
fmt.Println(full.Content)
```
