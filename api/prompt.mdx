The `prompt` package defines the `ChatTemplate` interface, which substitutes runtime variables into a predefined list of message templates and returns a `[]*schema.Message` ready to send to a `ChatModel`. The built-in `DefaultChatTemplate` supports three substitution engines: Python-style f-strings, Go's `text/template`, and Jinja2.

## Interface

### ChatTemplate

```go
type ChatTemplate interface {
    Format(ctx context.Context, vs map[string]any, opts ...Option) ([]*schema.Message, error)
}
```

#### Format

<ParamField path="ctx" type="context.Context" required>
  Request context.
</ParamField>

<ParamField path="vs" type="map[string]any" required>
  A map of variable names to values. Each key must match a placeholder used in the template. If a template placeholder is missing from `vs`, `Format` returns a runtime error — there is no compile-time safety check.

  Two types of values are accepted:
  - **Scalar values** (`string`, `int`, etc.) — substituted into message content strings.
  - **`[]*schema.Message` values** — inserted into the message list at a `MessagesPlaceholder` position.
</ParamField>

<ParamField path="opts" type="...Option">
  Call-time options. `DefaultChatTemplate` ignores all options; this parameter exists for custom `ChatTemplate` implementations that need per-call configuration.
</ParamField>

<ResponseField name="[]*schema.Message" type="[]*schema.Message">
  The rendered list of messages, in template order, with all variable placeholders replaced. Pass this slice directly to `ChatModel.Generate` or `ChatModel.Stream`.
</ResponseField>

<ResponseField name="error" type="error">
  Non-nil when a required variable key is missing from `vs`, the template syntax is invalid, or a `MessagesPlaceholder` value has the wrong type.
</ResponseField>

## DefaultChatTemplate

`DefaultChatTemplate` is the built-in `ChatTemplate` implementation. Create one with `FromMessages`:

```go
func FromMessages(formatType schema.FormatType, templates ...schema.MessagesTemplate) *DefaultChatTemplate
```

<ParamField path="formatType" type="schema.FormatType" required>
  The substitution engine to apply to message content strings.

  | Constant | Engine | Syntax example |
  |---|---|---|
  | `schema.FString` | Python f-string (via `pyfmt`) | `"Hello, {name}!"` |
  | `schema.GoTemplate` | Go `text/template` | `"Hello, {{.name}}!"` |
  | `schema.Jinja2` | Jinja2 (via `gonja`) | `"Hello, {{ name }}!"` |
</ParamField>

<ParamField path="templates" type="...schema.MessagesTemplate" required>
  One or more message templates. Each template is either:
  - A `*schema.Message` — rendered as a single message with variable substitution applied to `Content`.
  - A `schema.MessagesPlaceholder` — expands to a `[]*schema.Message` slice taken from `vs` at format time.
</ParamField>

## Template variable syntax

Placeholders in message content strings use the syntax of the chosen `FormatType`. Eino applies the substitution engine to the `Content` field of each `*schema.Message` template.

<Tabs>
  <Tab title="FString">
    Use curly-brace placeholders: `{variable_name}`.

    ```go
    template := prompt.FromMessages(schema.FString,
        schema.SystemMessage("You are a helpful assistant. Today is {date}."),
        schema.UserMessage("Answer in {language}: {question}"),
    )

    msgs, err := template.Format(ctx, map[string]any{
        "date":     "2026-04-22",
        "language": "English",
        "question": "What is Eino?",
    })
    ```
  </Tab>
  <Tab title="GoTemplate">
    Use Go `text/template` syntax: `{{.variable_name}}`.

    ```go
    template := prompt.FromMessages(schema.GoTemplate,
        schema.SystemMessage("You are a helpful assistant. Today is {{.date}}."),
        schema.UserMessage("Answer in {{.language}}: {{.question}}"),
    )

    msgs, err := template.Format(ctx, map[string]any{
        "date":     "2026-04-22",
        "language": "English",
        "question": "What is Eino?",
    })
    ```
  </Tab>
  <Tab title="Jinja2">
    Use Jinja2 syntax: `{{ variable_name }}`.

    ```go
    template := prompt.FromMessages(schema.Jinja2,
        schema.SystemMessage("You are a helpful assistant. Today is {{ date }}."),
        schema.UserMessage("Answer in {{ language }}: {{ question }}"),
    )

    msgs, err := template.Format(ctx, map[string]any{
        "date":     "2026-04-22",
        "language": "English",
        "question": "What is Eino?",
    })
    ```
  </Tab>
</Tabs>

<Warning>
  Variable keys present in the template but absent from `vs` produce a runtime error. Use consistent, well-documented key names across templates and callers to avoid silent mismatches.
</Warning>

## MessagesPlaceholder

`MessagesPlaceholder` inserts a slice of messages into the output at a specific position. Use it to inject conversation history or few-shot examples that are determined at runtime.

```go
func MessagesPlaceholder(key string, optional bool) schema.MessagesTemplate
```

<ParamField path="key" type="string" required>
  The key in `vs` that holds a `[]*schema.Message` value.
</ParamField>

<ParamField path="optional" type="bool" required>
  When `true`, a missing key in `vs` is silently treated as an empty message list. When `false`, a missing key is a runtime error.
</ParamField>

```go
template := prompt.FromMessages(schema.FString,
    schema.SystemMessage("You are a helpful assistant."),
    schema.MessagesPlaceholder("history", false),  // required: must be in vs
    schema.UserMessage("{question}"),
)

msgs, err := template.Format(ctx, map[string]any{
    "history": []*schema.Message{
        {Role: schema.User, Content: "What is Eino?"},
        {Role: schema.Assistant, Content: "Eino is a Go LLM framework by CloudWeGo."},
    },
    "question": "How do I add a retriever to a graph?",
})
```

## Options

`prompt.Option` carries implementation-specific configuration. `DefaultChatTemplate` defines no options of its own. When building a custom `ChatTemplate`, expose per-call parameters with `WrapImplSpecificOptFn`:

```go
func WithLocale(locale string) prompt.Option {
    return prompt.WrapImplSpecificOptFn(func(o *MyOptions) {
        o.Locale = locale
    })
}
```

Retrieve them inside your `Format` implementation with `GetImplSpecificOptions`:

```go
func (t *MyTemplate) Format(ctx context.Context, vs map[string]any, opts ...prompt.Option) ([]*schema.Message, error) {
    options := prompt.GetImplSpecificOptions(&MyOptions{Locale: "en"}, opts...)
    _ = options.Locale
    // ...
}
```

## Using prompts in a graph or chain

`ChatTemplate` typically precedes `ChatModel` in a graph. In a `Chain`, call `AppendChatTemplate`:

```go
import (
    "context"

    "github.com/cloudwego/eino/compose"
    "github.com/cloudwego/eino/components/prompt"
    "github.com/cloudwego/eino/schema"
)

func buildChain(ctx context.Context, chatModel model.BaseChatModel) {
    template := prompt.FromMessages(schema.FString,
        schema.SystemMessage("You are an expert in {domain}."),
        schema.MessagesPlaceholder("history", true),
        schema.UserMessage("{question}"),
    )

    chain := compose.NewChain[map[string]any, *schema.Message]()
    chain.AppendChatTemplate(template)
    chain.AppendChatModel(chatModel)

    runnable, err := chain.Compile(ctx)
    if err != nil {
        panic(err)
    }

    result, err := runnable.Invoke(ctx, map[string]any{
        "domain":   "distributed systems",
        "history":  []*schema.Message{},
        "question": "What is the CAP theorem?",
    })
    if err != nil {
        panic(err)
    }

    fmt.Println(result.Content)
}
```

In a `Graph`, connect the template node to the model node and use `compose.WithOutputKey` to convert upstream string output into the `map[string]any` that `Format` expects:

```go
graph := compose.NewGraph[map[string]any, *schema.Message]()
graph.AddChatTemplateNode("prompt", template)
graph.AddChatModelNode("model", chatModel)
graph.AddEdge("prompt", "model")
```

## Standalone usage

You can call `Format` directly without a graph:

```go
template := prompt.FromMessages(schema.FString,
    schema.SystemMessage("You are a concise assistant."),
    schema.UserMessage("Summarise this in one sentence: {text}"),
)

msgs, err := template.Format(ctx, map[string]any{
    "text": "Eino is a production-grade Go framework for building LLM-powered applications.",
})
if err != nil {
    panic(err)
}

// msgs is ready to pass to a ChatModel.
response, err := chatModel.Generate(ctx, msgs)
```
