Eino is an LLM application development framework for Go, built by the CloudWeGo team at ByteDance. It provides a complete toolkit — from low-level component abstractions to high-level agent patterns — letting you build production AI applications that follow Go idioms and handle streaming natively throughout.

## What Eino provides

Eino is structured around three layers that work together:

<Steps>
  <Step title="Components">
    Typed interfaces for LLM building blocks — `ChatModel`, `Tool`, `Retriever`, `Embedding`, `Prompt`, `Indexer`, and `Document`. Each interface has official implementations for OpenAI, Ollama, Claude, Gemini, Ark, Elasticsearch, and more in the [eino-ext](https://github.com/cloudwego/eino-ext) repository.
  </Step>
  <Step title="Composition">
    Connect components into graphs, chains, and workflows using `compose.NewGraph`, `compose.NewChain`, and `compose.NewWorkflow`. The framework handles streaming automatically — concatenating, merging, and fan-out of streams between nodes with no extra code.
  </Step>
  <Step title="Agent Development Kit (ADK)">
    Build AI agents with tool use, multi-agent coordination, interrupt/resume for human-in-the-loop, and ready-to-use patterns like DeepAgent and Supervisor. Agents can run as standalone processes or be composed as tools for other agents.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Type-safe generics" icon="shield-check">
    `Graph[I, O]`, `Chain[I, O]`, and `Workflow[I, O]` enforce input/output types at compile time
  </Card>
  <Card title="Automatic streaming" icon="wave-pulse">
    Stream processing is handled by the framework — components only implement what makes sense for them
  </Card>
  <Card title="Callback aspects" icon="bell">
    Inject logging, tracing, and metrics at `OnStart`, `OnEnd`, `OnError`, and stream lifecycle points
  </Card>
  <Card title="Interrupt/Resume" icon="pause">
    Pause agent execution for human input and resume from checkpoint — state is persisted automatically
  </Card>
  <Card title="DAG and Pregel modes" icon="sitemap">
    Graphs run as directed acyclic graphs or Pregel (with cycles), configurable per-graph
  </Card>
  <Card title="Graphs as tools" icon="plug">
    Expose any compiled graph as a tool for agents, bridging deterministic pipelines with autonomous behavior
  </Card>
</CardGroup>

<Note>
Eino requires Go 1.18 or later. Import the module with `github.com/cloudwego/eino`.
</Note>
